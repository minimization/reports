Fedora Minimization Objective

Minimization is a key goal of Fedora focusing on installation footprint of popular use cases in the Fedora ecosystem to make them more suitable for modern deployments such as IoT and cloud.

Fedora Minimization Docs

Feedback Pipeline Features

Use Case Focus

Use cases are groups of packages in a specific environment having a specific purpose. Compared to repositories or packages in general, this specific definition makes it easier to make decisions about their minimization.

Growth Prevention

Keeping use cases small is as important as making them small in the first place. Feedback Pipeline monitors use cases and helps maintainers identify sudden growth caused by changes in dependencies, build configurations, etc.

Quick Inspection

Feedback Pipeline provides dependency graphs for a quick inspection of dependency trees of use case installations. Clustering and interactive dependency highlights of any single package help discovering.
